RVU Relational HMI System v55.0

System Documentation — Rift Valley University, Bishoftu Campus

🏫 1. What Is This System?

The RVU Relational HMI System is a comprehensive, cloud-connected academic management platform for Rift Valley University, Bishoftu Campus. It covers the full student lifecycle — from registration and fee payment through grade management and GPA reporting.

  • Powered by Firebase Firestore for real-time cloud sync across all staff sessions.
  • All financial figures are in ETB (Ethiopian Birr).
  • Available in English, Afaan Oromoo, and Amharic.
  • Runs from a single HTML file — no installation required.
🔐 2. Who Uses This System?
  • Admin — Full control: all data, passwords, grade scale, registration rules, audit log.
  • Registrar — Open/close registration, permission matrix, enrolment oversight.
  • Cashier — Process payment requests, collect monthly tuition.
  • Finance Head — Set fee structures, view payment ledger, send fee notices.
  • Dept Head — Review student records, grade oversight, teacher uploads.
  • Student — View grades, GPA, print official report, submit payment requests.
🎓 3. Student Grade Portal — Step by Step
  • Step 1 — Enter your Student ID (e.g. 070/23) and press Next.
  • Step 2 — Select your Classification (Regular, Extension, Weekend…).
  • Step 3 — Select your Level of Education (Diploma, Degree, Masters, PhD).
  • Step 4 — Select your Department.
  • Step 5 — Select your Academic Year (Year 1 – Year 5).
  • Step 6 — Select your Semester. Only semesters with your data will appear.
  • Step 7 — View your results — subject, letter grade, credit hours, grade point, and GPA.
  • Step 8 — Click Print Your GPA Results for an official A4 report with registrar signature and verification code.
📦 4. Core System Modules
  • 🎓 General System: Course List, Grade Table, Student List, Credits, Student Grade Portal.
  • 🗄️ Database Viewer: Inspect all live tables; export full database as JSON.
  • 📋 Registration: New student intake, re-registration, level-separated DB (Diploma / Degree / Masters / PhD).
  • 📝 Registrar's Office: Permission matrix, registration open/close toggles.
  • 💰 Finance: Fee structure config, payment ledger, cashier approval queue, student payment portal.
  • 👩‍🏫 Teacher File: Upload instructor grade files (Excel/CSV) to update scores directly.
  • 🖥️ MS Office Suite: Built-in spreadsheet editor with Excel (.xlsx) export — accessible from the bottom toolbar.
📊 5. GPA Calculation & Academic Status

Formula: GPA = Σ(Grade Point × Credit Hours) ÷ Σ(Credit Hours)

A+/A = 4.0  |  B+ = 3.5  |  B = 3.0  |  C+ = 2.5  |  C = 2.0  |  D = 1.0  |  F = 0

  • 3.50 – 4.00 → EXCELLENT — Outstanding performance
  • 3.00 – 3.49 → VERY GOOD — Strong, consistent performance
  • 2.50 – 2.99 → GOOD — Solid academic standing
  • 2.00 – 2.49 → SATISFACTORY — Meets minimum requirements
  • 0.00 – 1.99 → FAIL — Below passing threshold
🔒 6. Security & Data Integrity
  • SHA-256 Password Hashing: All role passwords are hashed before cloud storage — never stored in plain text.
  • Audit Log: Every action (login, data change, payment approval) is time-stamped and recorded for institutional accountability.
  • Real-Time Sync: Firebase live listeners keep all open staff sessions updated simultaneously.
  • Verification Code: Printed GPA reports include an official code for authenticity checks at the Registrar's Office.
  • Each user is responsible for keeping their Student ID and credentials confidential.
⚙️ 7. Technical Information
  • Platform: Fully browser-based, powered by Firebase Firestore.
  • File Support: .XLSX, .XLS, .CSV for grade and student data upload.
  • Languages: English, Afaan Oromoo, Amharic (toggle + Google Translate).
  • Architecture: Single HTML file — no server or installation needed.
  • Technologies: HTML5, CSS3, Vanilla JS, Firebase Firestore, SheetJS, Google Translate API.
  • Bottom Toolbar: Quick-access bar with ⚙️ Settings, 🖥️ MS Office, 🖥️ Description, ⚖️ Rules, and 🏠 Home.
  • Navigation: 3-tier HMI — top pill-shaped panel selector → cut-corner sub-panel tabs → content area.

TERMS OF USE & USER RULES

RVU Relational HMI System — v55.0

📋 1. Authorised Use Only
  • This system is exclusively for registered students and authorised administrators of Rift Valley University, Bishoftu Campus.
  • Access by unauthorised persons is strictly prohibited and may be subject to disciplinary or legal action.
  • Each user is responsible for keeping their Student ID and any credentials confidential.
🔐 2. Administrator Responsibilities
  • Admin passwords must not be shared with students or non-authorised staff.
  • All data uploads (student lists, grades, scores) must be accurate and verified before submission.
  • Admins are fully responsible for any data modification performed under their credentials.
  • The password must be changed immediately if a security breach is suspected.
🎓 3. Student Usage Guidelines
  • Students may only view their own academic records using their registered Student ID.
  • Attempting to access, alter, or view another student's data is a serious violation.
  • Grade results displayed are for informational purposes. For official records, contact the Registrar's Office.
  • Printed GPA reports are unofficial unless stamped by the Registrar.
🚫 4. Prohibited Actions
  • Do not attempt to reverse-engineer, modify, or tamper with the system code.
  • Do not upload falsified or manipulated data files.
  • Do not share, publish, or distribute academic records of other students.
  • Do not use this system for any purpose outside its intended academic function.
🖨️ 5. Printed Documents
  • GPA result printouts are generated for personal reference only.
  • No part of any output from this system may be copied, reproduced, or submitted as an official document without the Registrar's authorisation.
  • The system watermark and signature line are included for integrity — altering them is prohibited.
⚠️ 6. Disclaimer
  • This system runs entirely in the browser. Data is synced live via Firebase Firestore.
  • The university is not liable for decisions made based solely on data viewed in this portal.
  • For grade disputes or corrections, contact your department or the Registrar's Office directly.
  • New and re-registration must be completed through the Registration panel accessed via the top navigation bar.
By using this system you acknowledge that you have read, understood, and agreed to these rules.
